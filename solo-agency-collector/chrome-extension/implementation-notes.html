2. Format link chung (đã chốt)

  • Dùng getHumanReadableText ⇒ mỗi <a> thành text(url) (mailto/tel có nhãn riêng). Cả 2 nhánh dùng chung hàm này để LLM thấy format nhất quán.

  • Bạn duyệt "bỏ nav/header/footer: Có". Nhưng <header>/<footer> còn được dùng cho header của từng post (chứa tên + link profile — thứ cần giữ).
  • Tinh chỉnh trong removeBoilerplateLandmarks:
    • Luôn bỏ: nav, role=navigation, role=banner, role=search (không bao giờ là nội dung post).
    • Chỉ bỏ header/footer/aside/role=contentinfo/role=complementary khi KHÔNG nằm trong article / role=article / role=feed → giữ header cấp post.

4. Dedup qua scroll

  • Tái dùng mergeAndRemoveDuplicateElementsOrdered (tách câu + URL, dedup câu trùng, LUÔN giữ URL, giữ thứ tự). Mỗi capture mới merge với text tích lũy của cùng URL.

5b. Phase 2 — final filter cho FEED (2026-06-22)

  • Merge prose cũ (mergeAndRemoveDuplicateElementsOrdered) làm nát format text(url) (sinh (. , .......) vì nó tách theo dấu chấm. → Feed KHÔNG dùng merge prose nữa; dùng finalFilterFeed (chỉ áp cho feed; website giữ Readability + prose-merge).
  • RÀNG BUỘC: tuyệt đối KHÔNG rule riêng cho nền tảng nào (không tên site, không denylist URL-path, không noiseWords). Tất cả phải generic.
  • Quy tắc cuối (DEFAULT_FILTER trong infinity_loops.js):
    • Bỏ cụm lặp: 1 token lặp liên tiếp ≥ repeatRun (=5) lần → xoá HẲN cả cụm (vd "Facebook Facebook…×5+").
    • Text giữa 2 link < 10 từ → bỏ — ĐÃ GỠ (cắt nhầm nhiều nội dung thật: comment ngắn, stats kèm link). Giờ giữ mọi unit trừ rỗng.
    • Bỏ chuỗi alnum ≥25 ký tự lẫn chữ+số không space (gibberishLen); URL được bảo vệ.
    • Strip token decoy (dropDecoyTokens): token ≥8 ký tự, có chữ+số, có chữ thường (loại acronym), đảo chữ↔số ≥3 lần → xóa (FB anti-scrape: DCsQ3O2.comNguyen, ptSoodnresfr5mc4170m). GIỮ EB2NIW/USDA/gpt-4o-mini/iPhone15/$5000/3h/9799.666.999.
    • Bỏ unit rỗng nghĩa: chỉ bỏ khi KHÔNG có từ thật (token toàn chữ ≥3) VÀ KHÔNG có URL path thật (cần CẢ HAI → không mất link post/profile/comment). Diệt dòng decoy thuần : haa3lyu1f0c9 ms1rf 2 /?__tn__=….
    • Dọn artifact ......., (. ), parens. Dedup theo label chuẩn hoá + URL path (bỏ query).
  • Kết quả trên collector.txt thật: 47k→6.7k ký tự (−86%), "Facebook" 286→0, URL 223→18; post/comment thật giữ nguyên.
  • Làm sạch URL (cleanUrlsInText, chỉ áp lên displayText; accumulator text giữ URL tuyệt đối để dedup không vỡ), 2 pass generic:
    • trimUrlBloat (=true, maxParamValueChars=40): bỏ query param có value DÀI & KHÔNG toàn số (blob tracking __cft__/__tn__); giữ param ngắn + ID số (comment_id, fbid, v=) → giữ định danh. Không cắt theo tên param (tránh platform-specific).
    • shortenSameDomainUrls (=true): URL cùng root-domain → còn path; URL khác domain giữ full (LLM biết link trỏ ra đâu).
    Tổng giảm ~77% so với raw (47k→10.9k trên collector.txt).
  • Residue: token decoy ngắn (comNguyen, Sedtorpnso5) dính liền vào text post thật → unit vẫn giữ vì có nội dung; chưa diệt vì generic-safe khó. Free-text không-link kiểu "Like Reply…" vẫn giữ theo đúng rule.

5c. Structural extraction từ raw DOM (2026-06-22, validate trên 2 file raw_html)

  • Feed KHÔNG còn dùng getHumanReadableText. Thay bằng extractStructured(html) trong infinity_loops.js — extraction theo ngữ nghĩa DOM (không class platform):
    • Strip non-content: script/style/noscript/svg/img/iframe/input/… + [aria-hidden="true"] (đây là nơi chứa "Facebook Facebook…" → diệt tận gốc, "Facebook" 286→0).
    • Strip chrome theo ARIA role: navigation/banner/contentinfo/complementary/dialog/toolbar/menu/combobox/search/progressbar/tablist/status… + nav/header/footer/aside. (engagement count trong dialog thông báo bị bỏ đúng — không phải post.)
    • Harvest engagement aria-label: ENGAGEMENT_RE = số + danh từ engagement phổ quát (reaction/comment/share/view/reply/answer). FB để count trong aria-label="194 reactions" + span số trần → harvest để giữ "2 reactions/1 reaction/3 replies…". Từ vựng phổ quát đa nền tảng, KHÔNG phải tên FB.
    • Serialize DOM-order: text node → text; <a> → text(href); block tag → newline. Parse trên CLONE (không đụng live page).
  • Decoy = tên account bị obfuscate (per-char span + CSS reorder). textContent (no CSS) ra chuỗi rác; looksLikeDecoyToken bắt 2 dạng: (a) chữ+số đảo ≥3 lần; (b) ≥10 ký tự có chữ-hoa-giữa. Tên đọc-được nằm ở anchor → vẫn giữ. Runtime note: cách diệt decoy chuẩn nhất là innerText live (CSS ẩn decoy + render tên đúng) — KHÔNG test được trên HTML tĩnh (file ko kèm CSS FB); hiện dùng heuristic.
  • Kết quả validate (node-html-parser, mirror logic): group_posts → sạch, đủ tên thật (Coi Dinh, Quynh Kumi, Hanh Lending), full post + comment + URL sạch + engagement (2/1/5/8 reactions, 3 replies), decoy 0, Facebook 0. newfeeds → sạch, có tên (stories); engagement 0 vì DOM lưu không có count trên post hiển thị (FB lazy-load), không phải lỗi thuật toán.
  • Precision decoy: unit-test 37 token thật (EB2NIW/USDA/gpt-4o-mini/iPhone15/McDonald/LinkedIn/$5000/3h/9799.666.999/tên…) → 0 false positive.
