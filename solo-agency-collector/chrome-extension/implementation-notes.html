Implementation notes - content filtering module

2026-06-22

1. Source Routing

  • Routing is based on platform, inferred from the URL by inferPlatform / inferPlatformInPage.
  • Feed-like sources such as Facebook, LinkedIn, Instagram, TikTok, X, Reddit, and YouTube use feed extraction. The collector keeps engagement hints and links, then deduplicates across scroll passes.
  • Website/article sources use Readability first to remove menus, top bars, recommendations, and page chrome before serialization.

2. Link Format

  • Human-readable text serializes anchors as text(url). Mail and phone links keep useful labels. The goal is a stable format that downstream LLMs can parse consistently.

3. Landmark Removal

  • Always remove nav, role=navigation, role=banner, and role=search.
  • Remove header, footer, aside, role=contentinfo, and role=complementary only when they are not inside an article/feed context. Post-level headers can contain author names and profile links that must be preserved.

4. Scroll Deduplication

  • Each new capture is merged with accumulated text for the same URL. URLs are preserved and repeated blocks are deduplicated while keeping source order.
  • Feed extraction avoids prose sentence-splitting because it can damage text(url) formatting and short comment/stat lines.

5. Manual Audit

  • The popup Capture now action injects collector helper scripts into the active tab, calls window.__collectorCapture(prevText), merges the result, and stores collector_audit in chrome.storage.local.
  • Show result opens audit.html for live review of merged text, character counts, URLs, and per-scroll captures.
  • The audit flow mirrors the automated collector scroll/capture loop and resets when the URL changes or the user presses Reset.

6. Feed Filtering Rules

  • Remove repeated token runs, long mixed letter/digit decoy strings, empty-meaning units, malformed parentheses, and generic artifacts.
  • Keep real URLs, post/profile/comment paths, short comments, engagement counts, names, prices, timestamps, and service details whenever they appear useful.
  • URL cleaning removes long tracking parameter values while preserving short IDs such as comment IDs, video IDs, and post IDs.
  • Same-domain URLs may be shortened to paths; cross-domain URLs remain full so downstream analysis can identify the destination.

7. Structural Extraction

  • Feed extraction prefers DOM-aware structure over raw whole-page text. It strips non-content nodes, common page chrome, and hidden decoys before serializing content.
  • Engagement labels are harvested from accessible labels when visible text does not include counts directly.
  • Decoy author tokens are detected by mixed letter/digit patterns and uppercase/lowercase anomalies. Live DOM text remains the preferred source when CSS hides decoys and reveals readable names.

8. Current Capture Routing

  • filtering.js is injected before infinity_loops.js and exposes globalThis.SocialHtmlFilter.
  • For feeds, SocialHtmlFilter.filterCurrentPage() runs first. If it returns usable text, the engine is filtering; otherwise the collector falls back to captureFeedText.
  • For web/article pages, Readability runs first. If Readability returns no useful article body, filtering handles index/homepage/list pages.
  • The capture envelope stays compatible: text fields are cleaner, URL candidates are preserved, and extraction_engine is an optional metadata field.

9. Known Follow-Ups

  • Some feeds may still choose contributor badges instead of real author names when the platform renders identity details unusually.
  • Some UI strings can still leak into content on non-English interfaces; expand language-specific UI filters cautiously without adding platform-specific rules.
  • A future two-step pipeline may use a lower-cost preprocessing model before the main analysis model to reduce token usage.
